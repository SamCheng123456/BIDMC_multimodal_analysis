# Git 项目管理说明

本仓库采用 Git 进行版本管理，提交记录按照课程项目的实际工作流程组织，便于复现实验步骤和追踪代码变更。

## 提交组织原则

1. `chore`：初始化项目结构、依赖文件和忽略规则。
2. `feat`：新增可运行功能，例如数据读取、预处理、全数据集分析、多模态融合和 GUI。
3. `docs`：整理论文、说明文档和图表材料。
4. `results`：提交可复现实验结果表和论文用图。

## 建议查看方式

```bash
git log --oneline --decorate
git show --stat
```
